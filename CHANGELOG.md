# Changelog

All notable changes to Dice Ex Machina.

## [Unreleased] — 全体バランス / 重さ対策 / ボス関門 / セット10種(Fusion分析)

独立2系統(Opus 4.8 ×2)の分析を突き合わせて実施:
- **重さ対策(主因)**: 毎ターンの同期フルセーブ(`JSON.stringify`+`localStorage`)が
  オートのカクつき主因。セーブを**約1.2sデバウンス＋タブ非表示/終了時にflush**へ。
- **700階ノンストップを止めるボス関門**:
  - 深層ボス(floor≥500)のHP/攻撃に「通常最強格(peer)」の床＋600超で非線形wallを追加
    (従来は深層ボスが通常敵より弱く関門にならなかった)。
  - ボス戦は**8ターン超で攻撃が毎ターン+30%加速(DPS関門)** → 火力不足だと突破不可。
  - **ボス突破後はオート自動停止** → 必ず武器集め/強化フェーズが発生。
- **敵★スケーリングの深層是正**: ★9以降(floor450+)は加算率を逓増させ、プレイヤーの
  乗算成長(職×セット×出目×★0.2×鍛冶)に追従(★8以下は不変)。
- **セット装備10種追加**(鉄壁/暗殺者/聖盾騎士/疾風/墓守/秘術士/終焉の使徒/巨盾兵/
  豪商/軍神の鎧)＋**新シナジー7種**(上位職含む)。火力偏重を避け防御/吸血/汎用に配分。
- floor<500・tier≤8・浅層解禁セットは不変で序盤〜中盤の回帰を最小化。test 124件緑。

## [Unreleased] — カジノに4号機の"中毒"システム / 4台&設定シャッフル / 台パン出禁 / コイン交換所

現代のパチンコ・パチスロで規制/禁止された射幸性の高い仕組みを分析し、ゲーム内に再現:
- **設定差**: 隠し設定(1-6)で機械割が変動（設定看破の沼）。**4台**用意し、**6時間ごとに**
  各台の設定を**シャッフル**（時間バケットで決定論的）。
- **天井**: ハマるほど近い救済（設定で540〜840G）。到達でBIG確定 → 天井狙いの中毒。
- **連チャンゾーン**: AT後の高確率(BIG/REG ×6・32G)で連チャン（ストック機的な出玉の波）。
- **青天井AT**: 出玉上限なし（コンプリート機能なし）。
- **台パン**: 見えづらい位置(右下)を叩くと**画面が揺れ＋鈍い衝撃音**。**10回**で**追い出し＝出禁**、
  **ボスを10体倒すまで**カジノ入店禁止（セーブに永続化）。
- **カジノコイン交換所**: 超高額のカジノコインで**セット武器**(🪙8000・所持ティア相当)や
  **転生ポイント**(🪙3000/pt・割高)と交換。
- test 121件グリーン／build 成功。

## [Unreleased] — 深層(700階+)のフリーズ / 読み込み遅延を修正

5並列調査で原因を特定。**主因はセーブの肥大化**だった。
- **図鑑データの無限肥大を停止**: ドロップ等で得た**手続き生成装備のID(`gen_*`/`setp_*`)を
  `discoveredItems` に溜め続けていた**。図鑑はキュレーション装備しか参照しないため完全な死蔵で、
  700階+では数千件に膨張 → 勝利毎の `progress` コピー＋`JSON.stringify` でフリーズ、起動時の
  パースで読み込みが遅延していた。今後はキュレーションIDのみ記録し、**既存セーブもロード時に
  自動で剪定**（バージョンを上げずに即解消）。
- **背景の再計算を排除**: `getWorldBackground` をワールド単位でメモ化。SVGのdampen正規表現＋
  URIエンコードがオート戦闘中に毎フレーム走っていた。
- **セット定義をメモ化**: `getSetDef` の手続きセット再構築（毎ターン複数回）をキャッシュ化。
- **アイコン描画を軽量化**: `ItemIcon` を `React.memo` 化＋`echo` 依存欠落を修正、アイコン
  キャッシュにLRU上限(400)を追加してメモリ膨張を防止。
- **所持上限の抜け穴を封鎖**: ショップ購入・残響報酬が `capInventory` を通さず150件を超えられた
  経路を修正。
- test 110件グリーン／build 成功。

## [Unreleased] — 手続き生成への移行（エンティティ非増殖）

6スロット化で「スロット×ティア」を静的に持つと装備数が倍々に増える問題を解消。

### Changed
- 通常ステ装備を **手続き生成**（`genItem(slot, tier)` / id `gen_<slot>_<tier>`）に移行。
  `getItemById` がIDから再構築するため、静的な生成装備リスト（旧366件）を廃止。
- `ITEMS` は **キュレーション装備（署名＋セット）のみ** の有限リストに。スロット数・階層が
  増えても登録数は一定、floor 1000+ もティアを保存せず表現可能。
- ドロップ/ガチャ/ショップを手続き生成に対応（限定・署名・セットはキュレーション側から混合）。
- **スマートドロップ**: 手続きドロップを空き/弱いスロットへ寄せ、6スロットでも収集が
  2倍にならないように（`gameStore` の `weakestSlot`）。
- 図鑑の装備収集率は **署名＋セットのみ** を対象（素材装備は対象外）。
