# セット装備

実装: `src/data/sets.ts`（定義と効果計算 `computeSetEffects`）、適用は `src/store/gameStore.ts`。

セット装備は Dice Ex Machina の重要要素。単なる数値アップではなく **ビルドを作る存在**。
6スロットに同じセットの装備を揃えると、2 / 4 / 6 部位で効果が段階的に解放される。

## 賭博師（🎲 gambler）

| 部位 | 効果 |
| --- | --- |
| 2 | リロール +1 |
| 4 | 出目1が2（通常攻撃）として扱われる |
| 6 | 出目6が2回発動する |

## 吸血鬼（🦇 vampire）

| 部位 | 効果 |
| --- | --- |
| 2 | 与ダメージの10%を吸血 |
| 4 | 出目4以上で追加吸血（+15%） |
| 6 | 全ての攻撃が30%吸血 |

## 処刑人（🪓 executioner）

| 部位 | 効果 |
| --- | --- |
| 2 | 出目5・6の威力上昇（+30%） |
| 4 | 攻撃時に追撃（+1ヒット） |
| 6 | 敵HP15%以下を即死（ボスを除く） |

## 神託（🔮 oracle）

| 部位 | 効果 |
| --- | --- |
| 2 | リロール時にHP回復 |
| 4 | 出目6の威力上昇（+60%） |
| 6 | ダイスを2個振り、高い方を使う |

## 実装メモ（決定論ダイスエンジンとの整合）

本作の出目は値で確定する決定論エンジンのため、セット効果は次の方針で適用している:

- **ダイス面書き換え**で表せるもの（賭博師4部位 1→2）は `diceModifiers` として
  `applyEquipmentModifiers` の最後段で合流。
- それ以外（吸血・高目ボーナス・追撃・即死・6の2回発動・リロール回復・2個振り）は
  `computeSetEffects` が返すフラグ/数値を `confirm()` / `reroll()` / `rollWithLuck()` で
  ダメージ・回復・出目に**加算的に**反映する。
- 「処刑（即死）」はボスには適用しない（ボス戦の興を削がないため）。
- ステータス系（賭博師2部位のリロール+1）は `passiveBonus()` に合算。

発動中のセット効果は装備画面（`EquipmentPanel`）に一覧表示され、図鑑（/collection の
「セット」タブ）で名前付きセットの効果を確認できる。

## 無限セット（手続き生成）

4つの名前付きセットに加え、**深層では新しいセット原型が手続き生成で無限に出現**する。

- セット効果は**効果プリミティブ**の組み合わせとして表現（`SetTierBonus`）。名前付き・手続き
  どちらも同じ仕組みで `computeSetEffects` が合成する。
- 手続きセットは `proceduralSetDef(n)`（key=`gset<n>`）が `n` から**決定論的**に名前・アイコン・
  2/4/6部位の効果を生成。`getSetDef(key)` で名前付き/手続きを解決。
- 出現解禁は `proceduralSetFloor(n) = 150 + n*150`（150階ごとに新セット）。
  `availableSetKeys(floor)` がその階で入手可能なセット一覧を返す。
- **セット装備もティア化**され、`genSetItem(key, slot, tier)`／id `setp_<key>_<slot>_<tier>` で
  深さに応じて無限にスケール（★も併用）。保存はIDのみで再構築可能。
- 名前付きセットも深さでドロップ・スケールするため、ビルドの更新は止まらない。

## セット × 職業シナジー

特定の **セット（≥N部位）× 職業** で追加効果が発動する（`SYNERGIES` / `computeSetEffects` に
`classId` を渡すと適用）。装備画面に「⚡ シナジー発動」として表示。

| 職業 | セット | 条件 | 効果 |
| --- | --- | --- | --- |
| 盗賊 | 賭博師 | 4部位 | リロール+1＆追撃 |
| 聖騎士 | 吸血鬼 | 4部位 | 全攻撃に+15%吸血 |
| 魔法使い | 神託 | 4部位 | 出目6の威力+40% |
| 狂戦士 | 処刑人 | 4部位 | 即死しきい値を20%に |
| 戦士 | 賭博師 | 4部位 | 出目5・6の威力+30% |
