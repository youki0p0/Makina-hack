## 1. 実装した内容

- 本編を一切壊さず、`arena` 名前空間で完全分離した別ゲームを新設（型・データ・ロジック・状態・UIすべて別ファイル、localStorageキーも別）。
- スマホ縦画面（max-w-md）の準備フェーズ → 全自動 3v3 バトル → 結果 → 次ラウンドのループ。
- 6フィールドが**技そのものを書き換える**フィールドシステム（数値補正ではない）。
- モンスター（戦うユニット, Dice Ex Machina 由来 9体 / 3色）と、新規オリジナルの**オペレーター（プレーヤーキャラ）5人**を分離。
- 色シナジー / タグシナジー / 集中・分散シナジー、オペレーター固有パッシブ。
- ショート(10勝/ライフ3)・ロング(15勝/ライフ5)モードと、モード別ランク（localStorage）。
- 戦闘ログ＋簡易バトルアニメ（フレーム再生・速度変更・スキップ）。

## 7. 5人会議によるレビュー（要約）

- 🎮 ディレクター: 「準備フェーズ＝カードを誰に集めるか」の意思決定が核として立っている。フィールド変質が毎ラウンドの再考を生むのが良い。採用。
- ⚖️ バランス: 序盤は快勝、9回戦以降で競る曲線に調整。集中はハイリスク・ハイリターンで成立。継続観察を推奨。
- 💻 リードエンジニア: tick制の決定論シミュ。本編と完全分離でき、再ハイドレートも安全。型エラー0・テスト緑。
- 👤 プレイヤー代表: 「選ぶ→配る→準備完了」の3手で分かりやすい。シナジーがその場で点灯するのが気持ちいい。スプライトが入れば一気に映える。
- 📈 ライブOps: ショート/ロングの別ランクで継続動機あり。モンスター・オペレーター・フィールドが拡張容易。

## 8. すぐ遊ぶ手順

```bash
npm install
npm run dev
# ブラウザで http://localhost:3000/arena
```

または本番デプロイ後、`/arena` に直接アクセス（メニューからは辿れない隠しゲーム。`/news` の #181 からも案内）。

遊び方: ①モード ②オペレーター ③モンスター3体 を選んで開始 → 毎ラウンド、提示カードをタップ→味方をタップで割り当て（技を1体に集めると「集中」） → 「準備完了！！」で観戦 → 勝てば次へ。
