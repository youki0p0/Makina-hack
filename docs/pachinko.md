# パチンコ「奈落海 / Abyss Marine」

カジノ内の**演出特化ミニゲーム**。「本物の賭博」ではなく、海物語系のような
“玉が流れて・図柄が揃って・出玉がジャラジャラ出る”気持ちよさの再現が目的。
海物語そのもののコピーはしない。**深海×歯車×ダイス×固有武器×神機マキナ**の世界観。

ルート: `/casino/pachinko`（カジノページからリンク）。

## 画面構図（スマホ縦・視線誘導）

盤面が**中央モニター(センター役物)を囲む「ひとつの台」**。実機同様、玉は天から
落ちて大きな役物の裏を通り、その直下の**ステージ**から出てくる。

```
所持玉 ／ 保留ランプ ／ モード
┌─────────────────────────┐
│  装飾釘   ▼MONITOR▼   装飾釘 │  ← 役物の窪みに図柄変動を重ねる（主役）
│ (左の縁)  固有武器3連  (右の縁)│
│            天釘             │
│        道釘・ステージ        │
│         ＼ ワープ ／         │
│            ヘソ            │  ← 入賞口（モニター直下）
└─────────────────────────┘
下部: 出玉 Canvas（払い出し玉）
発射 / オート発射 / 軽量モード / 演出ON-OFF
```

視線は **発射 → 役物上から落ちる玉 → ステージ＆ワープ → ヘソ入賞 → 中央モニターの
図柄変動 → 下部の大量払い出し** の順に流れ、視線は中央モニターに置いたままになる。

## 図柄（7種＝既存の固有武器＋神機マキナ）

絵柄は新規描画せず、既存アイテムの procedural ドット絵アイコンを流用する
（`src/data/items.ts` の固有武器 ＋ 神機マキナ）。

| id | 図柄 | 当たり | 規模 |
| --- | --- | --- | --- |
| 1 | 吸血の剣 | 吸血BONUS | 小当たり (150玉) |
| 2 | 毒牙の短剣 | 毒牙BONUS | 通常 (600玉) |
| 3 | 雷神の鎚 | 雷神BONUS | 通常+ (800玉) |
| 4 | 業火の剣 | 業火BONUS | 大 / 確変期待 (1200玉) |
| 5 | 氷結の槍 | 氷結BONUS | 大 / 継続期待 (1200玉) |
| 6 | 獄炎の薙刀 | 獄炎BONUS | 大当たり (1500玉) |
| 7 | 神機マキナ | 神機BONUS | **Jackpot (4000玉・虹発光)** |

**7以外でも3つ揃えば当たり。** 777 が最上位。

## 海物語風メカニクス

実機（海物語）の構造を踏まえて再現:

- **中央モニターが主役**。その**直下にヘソ(スタートチャッカー)**、さらに直上に
  **ステージ＆ワープ**を置く（`stageY` / `stageHalf` / `warpHalf`）。
- **ステージ＆ワープ**: モニター直下の舞台に玉が乗り中央へ転がる。中央ワープに
  乗ると**ヘソへ吸い込み濃厚**。乗れずこぼれた玉はほぼハズレ＝入賞のドキドキ。
- **道釘**で左上打ちの玉を中央へ流す。ヘソ直上の**命釘2本**が最後の関門
  （中央を通れるギリギリに絞り、ズレた玉は弾かれる）。
- **ステージ＝減衰振り子**（`stageRestoreK`/`stageDamp`）。中央への弱い復元力＋強い
  減衰で“泳ぐ”。中央通過が `warpVGate` 以下なら**ヌルッと吸い込み**、勢い余れば端から
  **惜しくこぼれる**。`stageMaxFrames` で滞留上限＝詰まり防止。
- **ヘソ入賞率 ≈20%**（命釘で中央に絞る）。`pocketW/stageHalf/warpVGate` と `launchBall`
  の `spread` で調整。**入賞でヘソに吸い込まれて縮む“間”**＋触覚(`navigator.vibrate`)。
- **当たり確率: 通常 1/30・確変中 0.40**（`reels.ts` `WIN_CHANCE`）。リーチ率 0.13。
  当たりを希少化＝価値を重く（回り続け、たまに当たり、確変で伸びる）。
- **保留(最大4)＝先読み**: 入賞時に1変動を内部抽選し、保留ランプの**色**(白→青→緑→赤→虹)
  で信頼度を示唆。消化前から期待が育つ。
- **確変/時短 = Makina Mode**: 4/5/6/7 当たりで突入し、残り回転を消化するまで
  **右打ち相当で自動高速回転**。**電サポ**でヘソ実効幅が広がる（`denchuBonusHalf`）。
  Jackpot は 100回転、その他の大当たりは 60回転。当たりで引き戻し（満タン補充）。

## 演出

- **リーチ＋スベリ＋沈黙**: 2つ揃いで最後の1つ待ち。第3停止前に**1コマ手前で止めて溜める
  スベリ**、神機マキナ群のときは**テンパイ前の沈黙を延長**（`reachExtra`）。
- **信頼度ピラミッド**: 弱(武器群/ダイス群/歯車群)→激熱(**神機マキナ群**＝魚群相当の全画面
  カットイン)。神機マキナ群はほぼ当たり時のみ＋**まれにガセ**（裏切りで興奮を作る）。`reels.ts`
  の `groupForWin/groupForMiss` で信頼度を設計。
- **昇格**: 333 →（歯車回転）→ 666 →（低確率）→ 777。
- **ラウンド制アタッカー**: 当たりは規模に応じたラウンド数(小2/通常4/大8/JP16)で
  出玉演出。`ROUND r/N` をモニター下に表示しつつ払い出し。
- **Jackpot**: 画面が虹色に発光、払い出し玉が最大量。

## 玉と物理

- **物理玉**: 実際に盤面を転がる。円-円の簡易反射のみ（Matter.js 不使用）。最大 `maxPhysicsBalls`。
- **払い出し玉**: 見た目用の粒子（物理なし）。Canvas にドット（四角）で描画、object pool で再利用。最大 `maxPayoutParticles`（Jackpot は `jackpotMaxPayoutParticles`）。
- **玉・釘はドット**で描画。ヘソ入賞はステージ経由中心で頻繁に起こる（デジタルは回り続ける）。

## 出玉（払い出し）

当たり時に一瞬で増やさず、`payoutQueue` を数秒かけて消化する。
玉カウンタが増えつつ、下部に払い出し玉が連続して流れる。

| 規模 | 玉数 | 時間 |
| --- | --- | --- |
| 小当たり | 150 | ~1秒 |
| 通常 | 600 | ~3秒 |
| 大当たり | 1500 | ~5秒 |
| Jackpot | 4000 | ~8秒・虹発光・最大量 |

## パフォーマンス

- `requestAnimationFrame` ＋ Canvas 描画。
- object pool で物理玉・払い出し粒子を再利用（GC 圧を抑制）。
- 非表示時（`visibilitychange`）は描画ループを停止。
- `prefers-reduced-motion` で自動的に軽量モード＋演出OFF。
- 手動の **軽量モード**（玉/粒子の上限と1フレーム放出を抑制）。

## 注意

所持玉はこのミニゲーム内のアーケード資源（演出用）で、セーブには保存しない
（`SAVE_VERSION` を上げず既存セーブへ影響を与えないため）。リロードで初期値に戻る。
