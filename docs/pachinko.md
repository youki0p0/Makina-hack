# 甘ダイス（甘デジ風パチンコ）

カジノ内の**演出特化ミニゲーム**。「本物の賭博」ではなく、海物語系のような
“玉が流れて・図柄が揃って・出玉がジャラジャラ出る”気持ちよさの再現が目的。
海物語そのもののコピーはしない。**深海×歯車×ダイス×固有武器×神機マキナ**の世界観。

ルート: `/casino/pachinko`（カジノページから「🎲 甘ダイスへ」）。

## 通貨（カジノコイン必須・1コイン=1玉）

カジノは**カジノコイン**で統一（スロット/BJ/甘ダイス/交換所）。甘ダイスは
**1コイン=1玉**で、発射ごとに-1コイン、払い出しでコイン加算（`addCoins`）。
スロットの**コイン⇄ゴールド換金**は据え置き＝コインの入口/出口はそこに集約。

- **収支設計（一撃特化・高ボラ）**：初当たりは激希少だが当たりは大きい。総RTP ≈ 2.8（スロットも兼ね合いで底上げ）。
  - **初当たり ≈ 1/392玉**（`WIN_CHANCE.normal = 1/130` × ヘソ約3割）。**ヘソ賞球 +2玉**（`BOARD.hesoPrize`・通常時のみ）で玉持ち。
  - **ボーナス＝枚数保証 × 保証ゲーム数**（`bonus.ts`）。図柄は演出のみ、出玉は振り分け表で決定（旧・確変STループを置換）:
    - 100枚/10G(w40) ・ 200枚/10G(w20) ・ 300枚/10G(w15) ・ 400枚/16G(w5)。
  - **継続ループ**：ラウンド終了時に継続抽選（10G≈78%/16G≈88%）＝**平均≈4.7連**（1チェイン平均≈850枚・最大数千枚）。
  - **出玉は大入賞口に入った玉で増える**（抜本変更）：当たり中は右打ち＝**発射無料**、`onAttacker` で1入賞 `ATTACKER_PRIZE` 枚ずつ加算し保証枚数まで。残りはラウンド終了でフラッシュ＝保証担保。当たり中は必ず純増・オート専用。
  - **当たり停止演出**：当たり目で止め **0.5〜1秒**（`REVEAL_MS`）“何が起きたか”を見せてから突入（スロット同様）。
  - **4台＋設定6段階（看破の沼）**：`casino.ts pachiMachineSettings`（スロットと別シード・6時間シャッフル）。
    設定倍率 `pachiSettingMult` は **0.70〜1.30（幅0.60＝スロット0.88〜1.28/幅0.40より広い）** で初当たり率に乗算。
    台選択で着席（隠し設定）、`pachiCeilingSpins` で天井 **420〜570回転**（高設定ほど浅い）。
  - **天井**：台の設定に応じた回転数ノーヒットで強制初当たり（`spinReels(…, forceWin)`）。ヘッダに回転数を表示。
  - **獲得枚数のその場表示**：ヘソ入賞(+2)・大入賞入賞(+`ATTACKER_PRIZE`)を役物の上に `+N` でふわっと表示（`coin-float`）。
- **スロット兼ね合い**：甘ダイスが甘いぶん、スロットの**特大上乗せ（鬼連チャン枠）を +500〜1200G に強化**（`casino.ts atRensho`／約1割のATで降る・期待値<1で収束）。
- BJ も**コイン**でベット（10/50/100）。

## 画面構図（スマホ縦・視線誘導）

盤面が**中央モニター(センター役物)を囲む「ひとつの台」**。実機（海物語）準拠で
**役物の裏は通らず**、玉は左右の**細い縦レーン**を流れる：**通常時=左打ち**（モニター
左のレーン→寄せ釘→中央ヘソ）、**当たり時=右打ち**（モニター右のレーン→右下の大入賞口）。

```
所持玉 ／ 保留ランプ ／ モード
┌─────────────────────────┐
│┃左 ┃   ▼MONITOR▼   ┃右 ┃│  ← 細い縦レーン（左=通常/右=当たり）
│┃打 ┃  固有武器3連   ┃打 ┃│
│ ＼寄せ釘        道釘 ┃ ┃ │
│        ステージ      [大入賞]│  ← 右打ちは右下アタッカーへ
│         ＼ ワープ ／         │
│            ヘソ            │  ← 入賞口（モニター直下）
└─────────────────────────┘
下部: 出玉 Canvas（払い出し玉）
発射 / オート発射 / 軽量モード / 演出ON-OFF
```

視線は **発射 → 役物上から落ちる玉 → ステージ＆ワープ → ヘソ入賞 → 中央モニターの
図柄変動 → 下部の大量払い出し** の順に流れ、視線は中央モニターに置いたままになる。

## 図柄（7種＝既存の固有武器＋神機マキナ）

絵柄は新規描画せず、既存アイテムの procedural ドット絵アイコンを流用する
（`src/data/items.ts` の固有武器 ＋ 神機マキナ）。

| id | 図柄 | 当たり | 規模 |
| --- | --- | --- | --- |
| 1 | 吸血の剣 | 吸血BONUS | 小当たり (150玉) |
| 2 | 毒牙の短剣 | 毒牙BONUS | 通常 (600玉) |
| 3 | 雷神の鎚 | 雷神BONUS | 通常+ (800玉) |
| 4 | 業火の剣 | 業火BONUS | 大 / 確変期待 (1200玉) |
| 5 | 氷結の槍 | 氷結BONUS | 大 / 継続期待 (1200玉) |

## 海物語風メカニクス

実機（海物語）の構造を踏まえて再現:

- **中央モニターが主役**。その**直下にヘソ(スタートチャッカー)**、さらに直上に
  **ステージ＆ワープ**を置く（`stageY` / `stageHalf` / `warpHalf`）。
- **打ち分けレーン（実機準拠）**: 役物の裏は通らず、左右の細い縦レーンを流れる。
  通常時=**左打ち**（`leftLaneX`→`chute*` 寄せスロープ→中央へ放出）、当たり時=**右打ち**
  （`rightLaneX`→右下の**大入賞口** `attacker*`）。`launchBall(board, rng, migiUchi)` で分岐、
  `migiUchi` は盤面の `denchu`(=Makina Mode) に連動。
- **ステージ＆ワープ**: モニター直下の舞台に玉が乗り中央へ転がる。中央ワープに
  乗ると**ヘソへ吸い込み濃厚**。乗れずこぼれた玉はほぼハズレ＝入賞のドキドキ。
- **寄せ釘ランプ→道釘**で左打ちの玉を中央へ流す。ヘソ直上の**命釘2本**が最後の関門
  （中央を通れるギリギリに絞り、左右を上下にずらして挟み込み詰まりを防止）。
- **ステージ＝減衰振り子**（`stageRestoreK`/`stageDamp`）。中央への弱い復元力＋強い
  減衰で“泳ぐ”。中央通過が `warpVGate` 以下なら**ヌルッと吸い込み**、勢い余れば端から
  **惜しくこぼれる**。`stageMaxFrames` で滞留上限＝詰まり防止。
- **ヘソ入賞率 ≈3割**（命釘で中央に絞る）。スロープ放出の勢い（`wobble`）と命釘 `gate`、
  `stageHalf/warpVGate` で調整。**入賞でヘソに吸い込まれて縮む“間”**＋触覚(`navigator.vibrate`)。
- **当たり確率: 通常 1/130・確変(演出)中 0.28**（`reels.ts` `WIN_CHANCE`）。リーチ率 0.13。
  初当たりを激希少化＝1撃を大きく（保証枚数×連チャン）。**ヘソ入賞で賞球+2**（玉持ち）。
- **保留(最大8)＝先読み**: 入賞時に1変動を内部抽選し、保留ランプの**色**(白→青→緑→赤→虹)
  で信頼度を示唆。消化前から期待が育つ。
- **ボーナス(RUSH) = 旧Makina Modeの置換**（`page.tsx` ＋ `bonus.ts`）: 初当たり→振り分けで
  {保証枚数, 保証G, 継続率} を決定→右打ち(発射無料)で大入賞口に出玉→保証G消化でラウンド終了→
  継続抽選でループ。出玉は **大入賞入賞ごと**に加算（`onAttacker`）し保証枚数を担保。
- **盤面は文字ラベルなし**: ヘソ＝光る吸い込み口（台形＋放射グロー＋きらめき）、大入賞＝開閉シャッター
  （右打ち中は金色に開いてグロー）、右打ち誘導はシェブロン(▸)で描画（`PachinkoBoard.tsx`）。

## 演出

- **リール＝横ドラム式（海物語準拠）**: 3リールを縦に3段（上/中/下）。各リールは図柄が
  **左→右へ横スクロール**し、判定は**中央列**（各段の中央コマ）。停止順は **上 → 下 → 中**。
  フェーズ：加速(3段同時)→定速→順次減速(上,下)→中リールの変則アクション。
  `PachinkoReels.tsx` の `animate`/tween で制御（軸を横にしただけで回転ロジックは共通）。
- **テンパイ＝上下が揃う**。中リールだけ変則停止:
  - **ノーマル**: トリガーの±1コマ手前から**カチ・カチとコマ送り**して本停止。
  - **SP発展(激アツ)**: **黒潮＝逆回転スロー**（行き過ぎて止まり、下→上へスロー逆回転）。
  - **ハズレ**は中リールがトリガーの**±1コマ**でズレて止まる（`reels.ts` の reach 目）。
- **海物語式リーチ演出の階段（信頼度ゲージ）**: `reels.ts` が1変動ごとに演出メタを抽選し、
  当たりほど強い演出が出やすい（＝信頼度）。`PachinkoReels.tsx` が段階表示する:
  - **SU予告（ステップアップ予告）** `su:0..4`: テンパイ前に段階点灯（青→緑→赤→金）。高ステップ濃厚。
  - **群予告** `group`: 弱(武器/ダイス/歯車群)→激熱(**神機マキナ群**＝魚群相当の全画面カットイン)。
  - **ノーマル→SP昇格** `reachKind`: SP発展で `spName`（テンパイ図柄にちなむ）のタイトル＋逆回転スロー。
    ハズレSPは**SP外し**（信頼度低め）。
  - **チャンスアップ** `chanceUp:0..3`: SP中に追加点滅。多いほど信頼度↑。
  - **プレミア** `premium`: 激レアの全画面「当確」（基本当たり時のみ）。
  - 生成は `suFor/chanceUpFor/spNameFor` と `groupForWin/groupForMiss` で信頼度を設計。
- **演出は“文字ではなくグラフィック/動き”で見せる**（`PachinkoReels.tsx` ＋ `globals.css` の `fx-*`）：
  - SU予告＝チャージpipの段階点灯 / 群予告＝図柄が左→右へ群れて流れる（魚群相当・makinaは金）。
  - SP発展/当たり確定＝テンパイ・当たり図柄を**衝撃リング付きでドンとカットイン**（`CutIn`）。
  - テンパイ/当確の瞬間＝**全画面フラッシュ**＋判定ラインの**スイープ光**。昇格＝図柄チェインがせり上がる。
  - プレミア＝**虹フラッシュ＋神機マキナの巨大回転**。キーフレームは `fxCutin/fxRing/fxFlash/fxSweep/fxHueSpin/fxCharge/fxRise/fxLine`。
- **昇格**: 333 →（歯車回転）→ 666 →（低確率）→ 777。
- **ラウンド制アタッカー**: 当たりは規模に応じたラウンド数(小2/通常4/大8/JP16)で
  出玉演出。`ROUND r/N` をモニター下に表示しつつ払い出し。
- **Jackpot**: 画面が虹色に発光、払い出し玉が最大量。

## 玉と物理

- **物理玉**: 実際に盤面を転がる。円-円の簡易反射のみ（Matter.js 不使用）。最大 `maxPhysicsBalls`。
- **払い出し玉**: 見た目用の粒子（物理なし）。Canvas にドット（四角）で描画、object pool で再利用。最大 `maxPayoutParticles`（Jackpot は `jackpotMaxPayoutParticles`）。
- **玉・釘はドット**で描画。ヘソ入賞はステージ経由中心で頻繁に起こる（デジタルは回り続ける）。

## 出玉（払い出し）

当たり時に一瞬で増やさず、`payoutQueue` を数秒かけて消化する。
玉カウンタが増えつつ、下部に払い出し玉が連続して流れる。

| 規模 | 玉数 | 時間 |
| --- | --- | --- |
| 小当たり | 150 | ~1秒 |
| 通常 | 600 | ~3秒 |
| 大当たり | 1500 | ~5秒 |
| Jackpot | 4000 | ~8秒・虹発光・最大量 |

## パフォーマンス

- `requestAnimationFrame` ＋ Canvas 描画。
- object pool で物理玉・払い出し粒子を再利用（GC 圧を抑制）。
- 非表示時（`visibilitychange`）は描画ループを停止。
- `prefers-reduced-motion` で自動的に軽量モード＋演出OFF。
- 手動の **軽量モード**（玉/粒子の上限と1フレーム放出を抑制）。

## 注意

所持玉はこのミニゲーム内のアーケード資源（演出用）で、セーブには保存しない
（`SAVE_VERSION` を上げず既存セーブへ影響を与えないため）。リロードで初期値に戻る。
