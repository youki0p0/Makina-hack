# CLAUDE.md

このリポジトリ（Dice Ex Machina / ダイスダンジョン）で作業するときの常時ルール。
詳しい開発指針はスキル `mobile-dice-hackslash`（`.claude/skills/mobile-dice-hackslash/SKILL.md`）を参照。

## 必ず守ること（要点）

- **1 Issue = 1 PR**。`main` へ直接 push しない。必ず作業ブランチ → ドラフト PR。
- 変更は**小さく**、既存仕様を壊さない。完了条件は `npm run build` が型エラー0で通ること。
- コアコンセプトを壊さない：**装備がダイスの出目効果を書き換える** / 戦闘操作は「リロール」「決定」の2つだけ / スマホ縦画面（最大幅 28rem）優先。
- 状態は `src/store/gameStore.ts` に集約。純粋計算は `src/lib/*`。localStorage には装備IDのみ保存。

## ★リリースノートの追記（毎回やる）

ユーザー向けに意味のある変更（新機能・調整・目に見える不具合修正など）を加えたら、
**同じ PR の中で必ずリリースノートにも追記する**。

- 追記先: `src/app/news/page.tsx` の `NOTES` 配列の**先頭**に新しいエントリを足す。
- 文体: **ちょっとおちゃめに**。既存エントリのトーンに合わせる（遊びの版名・絵文字・✦付き箇条書き）。
- エントリ形式:
  ```ts
  {
    date: "YYYY-MM-DD",          // 作業日
    tag: "v∞.NN「遊びの版名」",   // NN は対応する PR 番号などの目安
    emoji: "🎲",                 // 内容を表す絵文字1つ
    title: "ひとことで分かる見出し",
    items: ["何が変わったかを、ユーザー目線で1〜3行。"],
  },
  ```
- 内部リファクタ／ビルド版数の自動更新など、プレイヤーに無関係な変更は追記不要。
- メニュー最下部の「📜 リリースノート」リンク（`src/app/page.tsx`）から `/news` で確認できる。
